# 第一部分：我们为什么要做这个工具

## 1.1 背景

> 📌 **[插图建议：左侧列出痛点清单，右侧是 VoltDocs 图标，箭头指向"解决了"]**

随着公司业务向海外扩展，团队每周需要处理大量中英文文档——工程手册、安装说明、BOM 清单……

在没有这个工具之前，这件事情本身非常耗时，而且容易出错：

- **术语不统一：** "逆变器"，A 同事翻成 inverter，B 同事翻成 power converter，客户收到不同版本，专业形象受损
- **格式丢失：** 用通用翻译工具，粗体、表格、图片都没了，需要重新手动整理
- **效率低下：** 复制粘贴进翻译软件，再复制粘贴回来，每次至少半小时
- **记录分散：** 不同人用不同工具，翻译质量无法统一管控

## 1.3 三个核心设计理念

在整个工具的设计过程中，我们始终围绕三个原则：

### 理念一：格式必须完整保留

翻译完的文档，**格式要和原来一模一样**。粗体还是粗体，表格还是表格，图片还在原位，页眉页脚都不能丢。译文不是一段文字，而是一份可以直接交付的文档。

### 理念二：术语必须强制统一

公司有自己的专业词汇规范。工具会**强制 AI 按照我们的术语表翻译**，如果 AI 没有遵守，系统会自动标红提醒，让人工来核对。这是保证对外文件专业性的核心机制。

### 理念三：人工审校是最后一关

AI 不是万能的，它翻完的结果我们给它做质量检查（QA）。通过的直接下载，没通过的进入审校界面，让负责人人工修改确认后再导出。工具负责效率，人负责质量把关。

# 第二部分：工具的整体架构

## 2.1 工具由哪几部分组成

> 📌 **[插图建议：简洁的架构图，用色块和箭头表示，不要技术术语，只写功能名称]**
> 图示说明：用户通过浏览器访问 → 网页界面 → 后台服务处理文件 → 调用 AI 翻译 → 结果返回给用户

VoltDocs 由三个核心部分组成，就像一家工厂的流水线：

```
用户（浏览器）
    ↓ 上传文件、查看结果
网页界面（前台）
    ↓ 传递任务
后台服务（工厂）→ 解析文件 → 分批翻译 → QA 检查 → 生成译文
    ↓ 调用
AI 翻译引擎（Claude）
```

**网页界面（前台）**：你在浏览器里看到的页面，负责展示内容、接受操作。

**后台服务（工厂）**：你看不见的部分，负责实际处理文件——把文档拆解成句子，发给 AI 翻译，收回结果，再重新组装成原来的格式。

**AI 翻译引擎（Claude）**：我们使用的是 AWS（亚马逊云）提供的 Claude 模型，这是目前业界翻译质量最好的大语言模型之一。

## 2.2 翻译是怎么工作的

> 📌 **[插图建议：流程图，从"上传文件"到"下载结果"，每个步骤一个方块，箭头连接]**

很多人以为翻译就是"把整个文件扔给 AI，AI 吐出结果"，但实际上我们做了更精细的处理：

**第一步：拆解**

系统把文件拆解成一个个"段落"（我们叫 segment）。一个 Word 文档可能有几百个段落，每个表格单元格算一个，每个标题算一个。

**第二步：匹配术语**

在发给 AI 之前，系统先从术语库里找出这批段落里涉及的专业词汇，打包一起告诉 AI："这些词你必须这样翻。"

**第三步：分批翻译**

段落很多怎么办？不是一次全发，而是分成若干批，每批大约几十个段落，同时发给 AI，多批并行处理，速度更快。

**第四步：质量检查（QA）**

每个段落翻译回来以后，系统自动做 7 项检查（数字对不对、术语用了没有、格式标记有没有丢失……），不合格的标记出来。

**第五步：组装输出**

把所有翻好的段落重新组装回原来的文档格式——Word 还是 Word，Excel 还是 Excel，图片、表格、格式全部保留。

## 2.3 术语库是怎么工作的

> 📌 **[插图建议：左侧是术语库表格（中文/英文两列），右侧是翻译结果，箭头指向关键词高亮显示]**

术语库是这个工具里**最重要的质量保障机制**。

**什么是术语库？**

就是一张表格，记录公司规定的"某个中文词必须翻成哪个英文词"。比如：

| 中文术语 | 英文术语 |
|---------|---------|
| 逆变器 | inverter |
| 折叠支架 | folding bracket |
| 太阳能组件 | solar module |
| 安装手册 | installation manual |

**它是怎么生效的？**

翻译之前，系统会找出文档里涉及到的术语，然后直接告诉 AI："**你必须用这些词，不可以自己发挥。**"

如果 AI 没有按照术语表翻译，QA 系统会自动检测到，把这个段落标记为"未通过"，让审校人员检查。

**双向使用：**

同一条记录，中译英和英译中都会生效。维护一份就够，不用分别维护"中→英"和"英→中"两份。

---

## 2.4 QA 质量检查机制

> 📌 **[插图建议：七个图标排成一排，每个图标对应一个检查项，用绿色✓和红色✗示意通过/不通过]**

翻译完成后，每个段落都要经过 7 项自动检查：

| 检查项 | 说明 | 举例 |
|-------|------|------|
| 译文非空 | 确保 AI 没有漏译 | 某段落返回空白 → 标红 |
| 数字一致 | 原文里的数字在译文里必须出现 | 原文"AB-123 共 50 件"，译文没有"50"→ 标红 |
| 格式保留 | 粗体、斜体、删除线必须保留 | 原文有**重要**，译文变成普通文字 → 标红 |
| 长度合理 | 译文不能太短也不能太长 | 30 个字翻出 2 个字 → 可能漏译 → 标红 |
| 语言正确 | 中译英任务，结果应该是英文 | 大部分还是中文 → 标红 |
| 术语一致 | 术语库里的词必须用 | "逆变器"没有翻成"inverter"→ 标红 |
| 标点规范 | 英文句子不能用中文标点 | 英文翻译以"。"结尾 → 标红 |

**检查结果：**
- 全部通过 → 直接生成文件，可以下载
- 有不通过的 → 进入审校界面，人工处理

## 2.5 权限管理

> 📌 **[插图建议：三个人物图标，分别标注"超级管理员""管理员""普通用户"，用不同颜色区分，下方列出各自权限]**

工具支持三个级别的权限：

**超级管理员（Super Admin）**
- 可以做所有操作
- 额外权限：管理所有用户的角色（提升/降低别人的权限）

**管理员（Manager）**
- 可以翻译文档、转换文档
- 可以管理术语库（增删改导入）
- 可以查看操作日志

**普通用户（User）**
- 可以翻译文档、转换文档
- 不能管理术语库，不能管理用户

**登录方式：** 使用公司账号（与 Microsoft Teams 联动），不需要单独记密码。

## 3.3 功能一：文档翻译

> 📌 **[插图建议：截图，展示文档翻译页面，标注"上传区域""语言选择""开始翻译按钮""文件列表"四个区域]**

**支持格式：**
- Word 文档（.docx）
- Excel 表格（.xlsx）
- Markdown 文件（.md）

**操作步骤：**

**第一步：上传文件**

把文件拖入页面上的上传区，或者点击区域选择文件。**支持同时上传多个文件**，批量翻译。

> 📌 **[插图建议：截图，展示拖入文件后文件列表出现的状态]**

**第二步：选择翻译方向**

在右侧选择源语言和目标语言：

- 简体中文 → English（最常用）
- English → 简体中文

中间有一个"交换"按钮，可以快速切换翻译方向。

**第三步：点击"开始翻译"**

按钮上会显示待翻译的文件数量，点击后每个文件独立翻译，进度条实时更新。

> 📌 **[插图建议：截图，展示翻译进行中的进度条状态]**

**第四步：翻译完成后的操作**

翻译完成后，文件列表里每个文件会显示：

| 状态 | 含义 | 可以做什么 |
|------|------|-----------|
| 完成 ✅ | QA 全部通过 | 直接点「下载」|
| 需审校 🔵 | 有 QA 问题 | 点「审校」进入审校界面 |
| 失败 ❌ | 翻译出错 | 鼠标悬停查看原因 |

**如果有多个文件全部完成，可以点「打包下载全部」，所有文件打包成一个 .zip 下载。**

## 3.4 功能一（续）：审校界面

> 📌 **[插图建议：截图，展示审校弹框，标注"QA 问题描述""原文""译文编辑框""确认按钮"四个区域]**

点击「审校」后会弹出审校窗口，**只显示有 QA 问题的段落**（没问题的不需要看）。

**每条记录包含：**
- QA 问题描述（比如"术语未按术语表翻译：逆变器 → inverter"）
- 原文内容
- AI 翻译的初稿（可以直接编辑修改）
- 「确认」按钮（如果觉得原译文没问题，点确认保留）

**操作方式：**
- **修改译文：** 直接在译文框里改，改完颜色变绿，表示已处理
- **确认保留：** 觉得 AI 翻的没问题，点「确认」按钮
- **随时导出：** 无论有没有全部处理完，都可以点「导出译文」或「直接导出（含未处理项）」

> 💡 **提示：** 导出的文件和原文档格式完全一致，粗体、图片、表格都不会变。

## 3.6 功能三：术语库管理

> 📌 **[插图建议：截图，展示术语库列表页面，标注搜索框、启用开关、命中次数列]**

**谁能操作：** 管理员及以上权限

**核心作用：** 维护公司的标准专业词汇表，确保翻译时 AI 使用统一的译法。

### 添加单条术语

点击右上角「新增术语」，填写：
- 中文术语
- 英文术语
- 是否可用（默认开启）

### 批量导入

> 📌 **[插图建议：截图，展示批量导入预览弹框，显示新增/替换/跳过的分类]**

支持上传 `.xlsx` 或 `.csv` 格式的术语表文件。

**导入文件格式要求：**

| 列名 | 说明 |
|------|------|
| 中文术语（或 sourceTerm）| 原文 |
| 英文术语（或 targetTerm）| 译文 |

上传后系统会**预览导入结果**，区分哪些是新增、哪些是替换已有记录、哪些跳过（重复且没有变化），确认后再正式导入。

### 启用/禁用术语

每条术语右边有一个开关，可以**临时禁用某个术语**（比如某个词汇规范有争议，先暂停使用），而不是直接删除。

### 命中次数

> 📌 **[插图建议：截图，展示命中次数列，某些术语有数字，某些显示 "—"]**

列表里有一列"命中次数"，显示该术语在历史翻译任务里出现过多少次。这个数字帮助你了解哪些术语最常用，优先保证这些术语的准确性。

## 3.8 常见问题

> 📌 **[插图建议：FAQ 样式，问题和答案用不同背景色区分]**

**Q：翻译的速度快吗？**

一个普通的 20 页 Word 文档，通常 1-3 分钟内完成。文档越长、段落越多，时间越长。翻译过程中可以关闭页面，任务仍在后台运行。

---

**Q：翻译之后格式会不会变？**

不会。图片、表格、粗体、斜体、页眉页脚都会完整保留。这是我们区别于普通翻译工具的核心能力之一。

---

**Q：Excel 里的公式会被翻译掉吗？**

不会。只翻译文字内容，公式、格式、合并单元格全部保留。

---

**Q：术语表里没有的词，AI 会怎么翻？**

AI 会根据上下文自行翻译，这时候需要人工审校来把关。所以建议把常用的、有争议的词汇都录入术语库。

---

**Q：翻译任务失败了怎么办？**

在文件列表里鼠标悬停到"失败"状态，可以看到具体错误原因。常见原因：文件格式不对、文件加密无法读取。处理后重新上传即可。

---

**Q：审校完但没有点导出，进度会保存吗？**

会。关闭审校窗口后，已处理的内容仍然保留，下次打开继续审校即可。点导出时，未处理的项目会用 AI 的原始译文。

## 翻译一个文档的标准流程

```
1. 登录 VoltDocs
2. 点击「文档翻译」
3. 拖入文件 / 选择文件
4. 选择翻译方向（中文→英文 / 英文→中文）
5. 点击「开始翻译」
6. 等待进度完成
7. 文件状态变为"完成"→ 直接下载
   文件状态变为"需审校"→ 进入审校界面处理后导出
```

## 添加一条术语的标准流程

```
1. 点击「术语库」
2. 点击右上角「新增术语」
3. 填写中文术语 + 英文术语
4. 点击「添加」
5. 下次翻译自动生效
```
